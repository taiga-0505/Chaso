# planning
新機能の要望に対して、アーキテクチャ設計とタスクの細分化（TODOリスト作成）を行います。

## ルール (AI向けインストラクション)
1. **[行動制限]** 最初のターンでは絶対にソースコードの変更を行わないこと。
2. **[リサーチ]** ユーザーの要望に関連しそうな既存コードを `grep_search` や `view_file` で徹底的に調査し、現状のクラス設計や依存関係を把握すること。
3. **[設計立案]** 調査結果を元に、新しい機能をどこにどのように実装するかの設計を考えること。
4. **[タスク細分化]** 設計に基づき、実装タスクを `T-001`, `T-002` のような細かいステップに分解すること。
5. **[承認待ち]** プランを提示した後は、必ずユーザーからのフィードバックと承認（Goサイン）を待つこと。

## 出力フォーマット
```markdown
# 🏗️ アーキテクチャ設計と実装プラン

## 1. 現状の分析と設計方針
## 2. 実装タスクの細分化
- [ ] **T-001:** [ファイル名] - [やること]
- [ ] **T-002:** [ファイル名] - [やること]
## 3. 懸念事項・確認事項
```
