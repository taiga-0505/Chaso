# review
指定されたC++ファイルの品質を評価し、パフォーマンス、安全性、可読性の観点からレビューを行い、モダンなC++へのリファクタリング案を提示します。

## ルール (AI向けインストラクション)
1. **[解析]** 指定されたファイル（またはActive Document）を `view_file` で読み込み、構造を把握すること。
2. **[安全性チェック]** 以下のアンチパターンがないか厳格にチェックすること。
   - 生ポインタ(`*`)の危険な使用、所有権の曖昧さ
   - メモリリーク、解放忘れのリスク
   - スレッドセーフティの問題
3. **[パフォーマンスチェック]** 以下の非効率な処理がないかチェックすること。
   - 不要な値渡し（`const Type&` でないもの）
   - コピーの多発
   - キャッシュラインを意識していないデータ構造
4. **[モダンC++]** C++20（または標準）を意識し、スマートポインタ、`auto`、範囲ベースforループ、Conceptなどの活用を提案すること。

## 出力フォーマット
```markdown
# 🔎 C++ コードレビュー結果

## 1. 全体評価
## 2. 指摘事項と改善案
### 🔴 重要度: 高（安全性・バグのリスク）
- [問題点] -> **改善案:** [直し方]
### 🟡 重要度: 中（パフォーマンス・モダン化）
- [問題点] -> **改善案:** [直し方]
## 3. 具体的なリファクタリング例 (Diff)
```
