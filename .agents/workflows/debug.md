# debug
ビルドエラーや実行時クラッシュのログを受け取り、素早く原因を特定し、修正手順を導き出します。

## ルール (AI向けインストラクション)
1. **[仮説思考]** ユーザーから提示されたエラーログや現象を見て、考えられる原因の「仮説」を必ず3つ立てること。
2. **[自律的リサーチ]** 仮説を検証するために、`grep_search` や `view_file` を使って関連するソースコードを調査すること。
3. **[原因特定]** リサーチの結果から、最も可能性の高い根本原因を絞り込むこと。
4. **[解決策の提示]** 原因を取り除くための具体的な修正ステップを提示すること。

## 出力フォーマット
```markdown
# 🚑 エラー原因調査レポート

## 1. 考えられる原因（仮説3つ）
1. **仮説A:** [内容]
2. **仮説B:** [内容]
3. **仮説C:** [内容]
## 2. 調査結果
## 3. 根本原因
## 4. 解決のための修正ステップ
- [ ] **Step 1:** [ファイル名] の修正
- [ ] **Step 2:** [その他の手順]
```
