# CLI ビルド＆実行環境の構築

## 概要

Visual Studio を起動せずに、コマンドライン（CLI）または VS Code から **ビルド** → **実行** を一発で完結できるスクリプト環境です。

## 方法1: VS Code のタスク機能を使う（超おすすめ）

キーボードで **`Ctrl + Shift + B`** を押すだけです。（または「ターミナル」メニューから「ビルド タスクの実行...」を選択）

- `🚀 Run Debug (ビルド＆実行)` …… デフォルト。デバッグ構成で起動します。
- `🚀 Run Release` …… リリース構成で起動します。
- `🔨 Build Only (Debug)` …… 実行せずにビルドだけ行います。

## 方法2: ターミナルから手動で実行する

プロジェクトのルート（一番上のフォルダ）から、以下のコマンドをコピー＆ペーストして実行してください。

### 基本（Debug 構成で実行）

```powershell
powershell -ExecutionPolicy Bypass -File "scripts/build/build_and_run.ps1" -Configuration Debug
```

### Release 構成で実行

```powershell
powershell -ExecutionPolicy Bypass -File "scripts/build/build_and_run.ps1" -Configuration Release
```

### オプションの追加（手動ビルド用）

**ビルドのみ（実行しない）**
```powershell
powershell -ExecutionPolicy Bypass -File "scripts/build/build_and_run.ps1" -Configuration Debug -BuildOnly
```

**実行のみ（ビルドをスキップ）**
```powershell
powershell -ExecutionPolicy Bypass -File "scripts/build/build_and_run.ps1" -Configuration Debug -RunOnly
```

## ログについて

- **アプリ実行時のログ**: `logs/app/` フォルダに保存されます。バッファリング問題を解消し、リアルタイムでファイルに書き出されるようになっています。
- **ビルドのログ**: `logs/build/` フォルダに保存されます。ビルドの成否に関わらず、詳細な情報が記録されます。
